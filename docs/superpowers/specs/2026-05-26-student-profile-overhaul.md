# Student Profile Overhaul — Design Spec

**Goal:** تحويل ملف الطالب من صفحة عرض بسيطة إلى لوحة تحكم كاملة تجيب على الأسئلة اليومية فوراً: هل متأخر؟ هل استلم؟ هل تم التواصل معه؟ كم دفع ومتى؟

---

## 1. شريط الحالة الفورية (Status Bar)

**أين:** أعلى صفحة تفاصيل الطالب مباشرةً، قبل البطاقة الرئيسية.

**ما يعرض:** أربع شارات حالة:

| الشارة | متى تظهر | اللون |
|--------|----------|-------|
| متأخر في السداد | `totalFees - paidAmount > 0` وعمر الرصيد > 30 يوم من تاريخ القبول | أحمر |
| منتظم | لا توجد متأخرات | أخضر |
| الكتب: مُسلَّمة ✓ | يوجد delivery order بحالة `delivered` يحتوي أصناف كتب | أخضر |
| الكتب: لم تُسلَّم | عكسه | رمادي/أصفر |
| الزي: مُسلَّم ✓ | نفس المنطق للزي | أخضر |
| الزي: لم يُسلَّم | عكسه | رمادي/أصفر |

الحكم على "كتب" و"زي" يستخرج من جدول `DeliveryOrder` الموجود — لا جداول جديدة.

---

## 2. التعديل السريع من داخل الملف (Quick Edit)

**المشكلة:** تحديث رقم التليفون أو نقل الطالب لفصل آخر يتطلب العودة لقائمة الطلاب.

**الحل:** ثلاثة إجراءات inline داخل بطاقة معلومات الطالب:

### أ. تحديث رقم الهاتف
- زر ✏️ بجانب الرقم مباشرةً
- يفتح input مكانه، حفظ بـ Enter أو زر ✓، إلغاء بـ Escape
- يرسل `PATCH /api/students/:id` بـ `{ guardianPhone }`

### ب. نقل الفصل
- زر ✏️ بجانب "الصف / الفصل"
- Dialog بسيط: حقل الصف + حقل الفصل فقط
- يرسل `PATCH /api/students/:id` بـ `{ grade, className }`

### ج. صورة الطالب
- الدائرة الحالية (أيقونة مستخدم) تصبح قابلة للنقر
- يفتح `<input type="file" accept="image/*">`
- يرفع الصورة لـ `POST /api/students/:id/photo` (endpoint جديد)
- الخادم يخزن الصورة كـ base64 في حقل `photoUrl` الموجود (مناسب للحجم المتوقع لصور الهوية)

## 3. سجل التواصل (Contact Log) — تبويب جديد

**المشكلة:** لا يوجد تتبع لمن اتصل بولي الأمر ومتى ونتيجة المكالمة.

**الحل:** تبويب "سجل التواصل" بين التبويبات الموجودة.

### نموذج إضافة اتصال
- التاريخ (يملأ تلقائياً بتاريخ اليوم، قابل للتعديل)
- الملاحظة (textarea: "وعد بالدفع الأسبوع القادم"، "لا يرد"، ...)
- النتيجة: dropdown — تم التواصل / لم يرد / وعد بالسداد / مدفوع بعد التواصل

### العرض
- قائمة زمنية عكسية (الأحدث أولاً)
- كل سجل: التاريخ + اسم المستخدم الذي سجّل + الملاحظة + النتيجة
- لا حذف (سجل أثري)

### قاعدة البيانات
جدول جديد `StudentContactLog`:
```
id          String  @id @default(uuid())
studentId   String
date        String  (YYYY-MM-DD)
notes       String
outcome     String  (contacted | no_answer | promised | paid_after)
createdBy   String  (userId)
createdAt   DateTime @default(now())
```

Routes جديدة:
- `GET /api/students/:id/contacts`
- `POST /api/students/:id/contacts`

---

## 4. وضوح المديونية السابقة

**المشكلة:** المديونية من السنة السابقة (`arrearsFees`) موجودة لكن لا يُعرف من أين جاءت.

**الحل:** في بطاقة الملخص المالي:
- إذا كان `arrearsFees > 0`: عرض تفصيلي "مديونية العام السابق: X" مع رابط للتبويب "السجل المالي للسنوات"
- تبويب "السجل المالي للسنوات" يُظهر لكل سنة: الرسوم الكاملة، المدفوع، المتبقي، الحالة

---

## 5. تحسينات صفحة القائمة (`/students`)

**إضافة فلاتر للشريط الأعلى:**
- فلتر الحالة (نشط / غير نشط / متخرج / منقول)
- فلتر الصف (يُملأ ديناميكياً من بيانات الطلاب المحملة)

**إضافة عمود "آخر دفعة" في الجدول:**
- يعرض تاريخ آخر دفعة لكل طالب (من `payments` المحملة في الـ store)

---

## ما لا يُبنى (خارج النطاق)

- نظام إشعارات تلقائية لولي الأمر
- تصدير Excel / PDF (يمكن إضافته لاحقاً)
- تعديل الرسوم من داخل ملف الطالب (يظل في سير عمل القبول)
- نظام إدارة ملفات متكامل (المستندات تُرفع كصور base64 مؤقتاً)
